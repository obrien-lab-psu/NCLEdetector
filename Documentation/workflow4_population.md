# Workflow 4: Population-Level Detection of Misfolding Involving Native Entanglements

← [Back to Master Index](index.md)

---

## Goal

Using a proteome-wide experimental dataset (e.g., differential LiP-MS), test whether proteins or residues in regions containing native entanglements are statistically more likely to show conformational change signals. Identify subpopulations of proteins with the strongest enrichment using Monte Carlo selection.

---

## Typical runtime

| Step | Runtime |
|------|---------|
| Native NCLEs for all proteins | Hours to days (run Workflow 1 in batch) |
| LiP-MS data processing | Minutes to hours |
| Logistic regression | Minutes |
| Monte Carlo (one run) | Hours |
| Monte Carlo (multiple runs) | Days |

---

## Conceptual overview

This workflow is **independent of the trajectory-based workflows** (2 and 3). It operates at the proteome level, not on a single protein.

The logic is:

1. Run Workflow 1 for every protein in the proteome dataset to identify its native entanglements.
2. Process experimental LiP-MS data to get residue-level conformational change signals.
3. Define confounders (e.g., amino acid identity, solvent accessibility).
4. Build a residue-level data table for each protein.
5. Fit logistic regression models to test for enrichment of conformational change in entangled regions.
6. Use Monte Carlo selection to identify protein subpopulations with the strongest enrichment.

## Step 1. Compute native NCLEs for all proteins in the dataset

For every protein observed in the high-throughput experiment, run the Workflow 1 pipeline in batch.

### Batch script example

```python
import os
from EntDetect.gaussian_entanglement import GaussianEntanglement
from EntDetect.clustering import ClusterNativeEntanglements, ClusterNativeEntanglements

DATASTORE = "/scratch/ims86/EntDetect_Datastore"
TESTDIR   = "/storage/group/epo2/default/ims86/git_repos/EntanGoPy/TestingGrounds"
OUTDIR    = f"{DATASTORE}/outputs/workflow4"

pdb_dir  = f"{DATASTORE}/user_input/proteome_structures/AF"     # use AF/ for AlphaFold; EXP/ for experimental
model    = "AF"                            # "AF" or "EXP"
ncle_dir = f"{OUTDIR}/nativeNCLE_all"

ge        = GaussianEntanglement(g_threshold=0.6, density=0.0, Calpha=False, CG=False)
clustering = ClusterNativeEntanglements(organism="Ecoli")

# Process each structure
for pdb_file in sorted(os.listdir(pdb_dir)):
    if not pdb_file.endswith(".pdb"):
        continue

    pdb_path = os.path.join(pdb_dir, pdb_file)
    gene_id  = pdb_file.replace(".pdb", "")  # UniProt accession as ID

    native_outdir    = f"{ncle_dir}/{gene_id}/Native_GE"
    hq_outdir        = f"{ncle_dir}/{gene_id}/Native_HQ_GE"
    clustered_outdir = f"{ncle_dir}/{gene_id}/Native_clustered_HQ_GE"
    feature_outdir   = f"{ncle_dir}/{gene_id}/Native_clustered_HQ_GE_features"

    os.makedirs(native_outdir, exist_ok=True)

    # Step 1a: detect native entanglements
    ge.calculate_native_entanglements(clean_pdb=pdb_path, outdir=native_outdir, ID=gene_id)

    # Step 1b: filter for high-quality entanglements
    ncle_file = f"{native_outdir}/{gene_id}_GE.txt"
    if not os.path.exists(ncle_file):
        print(f"  No NCLE file found for {gene_id}, skipping HQ filter")
        continue

    os.makedirs(hq_outdir, exist_ok=True)
    ge.select_high_quality_entanglements(
        ncle_file, pdb_path,
        outdir=hq_outdir, ID=gene_id, model=model
    )

    # Step 1c: cluster into representative entanglements
    hq_file = f"{hq_outdir}/{gene_id}.csv"
    if not os.path.exists(hq_file):
        continue

    os.makedirs(clustered_outdir, exist_ok=True)
    clustering.Cluster_NativeEntanglements(
        hq_file, outdir=clustered_outdir, outfile=f"{gene_id}.csv"
    )
```

> **For large-scale runs:** Submit this as an array job on your HPC cluster, one job per protein. Runtime scales with protein length.

### Note on experimental structures with residue mapping files

For structures in `$DATASTORE/user_input/proteome_structures/EXP/`, each PDB is paired with a `*_resid_mapping.txt` file that maps structure residue numbering to canonical UniProt sequence positions. When present, pass the mapping file to `select_high_quality_entanglements` to restrict NCLEs to canonical residues.

## Step 3. Define confounding variables

Before regression, decide which covariates to include as confounders. Common choices:

| Confounder | Rationale |
|------------|-----------|
| Amino acid identity (`AA`) | Certain residues are intrinsically more LiP-reactive |
| Solvent accessibility (`SASA`) | Exposed residues are more accessible to protease |
| Protein length | Longer proteins have more opportunities for detection |

Define these in your residue-level data table (Step 4).

---

## Step 4. Build residue-level data frames

For each protein, construct a table where each row is one residue:

| Column | Meaning |
|--------|---------|
| `residue_index` | Residue number in the canonical sequence |
| `cut_C_Rall` | 1 if the residue shows a significant LiP-MS signal; 0 otherwise |
| `region` | 1 if the residue belongs to an entangled region; 0 otherwise |
| `AA` | Single-letter amino acid code |
| `SASA` | Solvent-accessible surface area (from native structure or simulation) |
| `mapped_resid` | Canonical UniProt residue index (used for masking incomplete coverage) |
| `gene` | Gene / UniProt identifier |
| `uniprot_length` | Full canonical protein length |

Save each protein's table to a directory; all per-protein files go in one folder, one file per gene.

## Step 5. Run proteome-level logistic regression

Use `ProteomeLogisticRegression` to model the association between entangled regions and experimentally observed conformational change, while controlling for confounders.

```python
from EntDetect.statistics import ProteomeLogisticRegression

DATASTORE = "/scratch/ims86/EntDetect_Datastore"
OUTDIR    = f"{DATASTORE}/outputs/workflow4"

dataframe_files = f"{OUTDIR}/residue_dataframes"   # directory of per-gene data files
gene_list       = f"{OUTDIR}/gene_list.txt"         # plain-text file, one gene per line
outdir          = f"{OUTDIR}/population_modeling"
ID              = "Ecoli_noChaperones"
reg_formula     = "cut_C_Rall ~ AA + region"

ProtRegression = ProteomeLogisticRegression(
    dataframe_files=dataframe_files,
    outdir=outdir,
    gene_list=gene_list,
    ID=ID,
    reg_formula=reg_formula
)

ProtRegression.load_data(
    sep='|',
    reg_var=['AA', 'region'],
    response_var='cut_C_Rall',
    var2binarize=['cut_C_Rall', 'region'],
    mask_column='mapped_resid'
)

reg_df = ProtRegression.run()
print(reg_df)
```

### Understanding the `reg_formula` argument

The formula follows R-style syntax:

```
response ~ covariate1 + covariate2 + ...
```

For example:
- `"cut_C_Rall ~ AA + region"` tests whether `region` (entangled vs non-entangled) is associated with LiP-MS signal (`cut_C_Rall`), controlling for amino acid identity (`AA`).
- Add `SASA` to the formula if you want to additionally control for solvent accessibility.

### Interpreting the regression output

For the `region` coefficient:

$$\text{Odds Ratio} = e^{\beta_{\text{region}}}$$

| Result | Interpretation |
|--------|---------------|
| $\beta > 0$, p < 0.05 | Entangled residues are more likely to show LiP-MS signal |
| $\beta < 0$, p < 0.05 | Entangled residues are less likely to show LiP-MS signal |
| p ≥ 0.05 | No significant association detected |

> **Critical:** If you run this test across multiple conditions, data subsets, or experimental replicates, apply **FDR correction** (e.g., Benjamini–Hochberg) to the resulting p-values to control false positives.

## Step 6. Monte Carlo selection of high-risk protein subpopulations

Identify protein subsets whose entanglements are most strongly associated with experimental signals.

```python
from EntDetect.statistics import MonteCarlo

DATASTORE = "/scratch/ims86/EntDetect_Datastore"
OUTDIR    = f"{DATASTORE}/outputs/workflow4"

dataframe_files = f"{OUTDIR}/residue_dataframes"
gene_list       = f"{OUTDIR}/gene_list.txt"
outdir          = f"{OUTDIR}/monte_carlo"
ID              = "Ecoli_noChaperones"
reg_formula     = "cut_C_Rall ~ AA + region"

MC = MonteCarlo(
    dataframe_files=dataframe_files,
    outdir=outdir,
    gene_list=gene_list,
    ID=ID,
    reg_formula=reg_formula
)

MC.load_data(
    sep='|',
    reg_var=['AA', 'region'],
    response_var='cut_C_Rall',
    var2binarize=['cut_C_Rall', 'region'],
    mask_column='mapped_resid',
    ID_column='gene',
    Length_column='uniprot_length'
)

MC.run(encoded_df=MC.data, ID_column='gene')
```

### Expected outputs

| File | Contents |
|------|----------|
| Objective function statistics | Odds-ratio trajectory over Monte Carlo steps |
| Per-group bin statistics | Odds-ratio and significance for each protein group at each step |

## Step 7. Rank groups after convergence

After the Monte Carlo simulation reaches steady state, rank groups by their average odds ratio over the final 100 steps:

```python
import pandas as pd
import numpy as np

DATASTORE = "/scratch/ims86/EntDetect_Datastore"
OUTDIR    = f"{DATASTORE}/outputs/workflow4"

# Load Monte Carlo step statistics (filename will depend on your ID and run)
mc_stats_file = f"{OUTDIR}/monte_carlo/{ID}_bin_stats.csv"   # adjust filename as needed
mc_df = pd.read_csv(mc_stats_file)

# Take the last 100 steps
last_100 = mc_df[mc_df['step'] >= mc_df['step'].max() - 99]

# Average odds ratio per group and rank
group_or = (
    last_100.groupby('group')['odds_ratio']
    .mean()
    .reset_index()
    .sort_values('odds_ratio', ascending=False)
)

print("Top-ranked protein groups:")
print(group_or.head(10))
```

The top-ranking group contains proteins most strongly associated with experimentally observed conformational changes in entangled regions.

## Step 8. Refine candidate selection across repeated runs

To reduce sensitivity to Monte Carlo stochasticity:

1. Run **multiple independent Monte Carlo simulations** (recommend ≥ 10 runs).
2. For each protein, count how many runs placed it in the top odds-ratio group.
3. Keep only proteins appearing in the top group in **more than 70%** of runs.

```python
import os
import pandas as pd

DATASTORE = "/scratch/ims86/EntDetect_Datastore"
OUTDIR    = f"{DATASTORE}/outputs/workflow4"

mc_run_dirs = [f"{OUTDIR}/monte_carlo_run{i}" for i in range(1, 11)]
protein_counts = {}

for run_dir in mc_run_dirs:
    # Load the top-group membership from this run
    top_group_file = f"{run_dir}/top_group_members.csv"   # adjust as needed
    if not os.path.exists(top_group_file):
        continue
    top_members = pd.read_csv(top_group_file)
    for gene in top_members['gene']:
        protein_counts[gene] = protein_counts.get(gene, 0) + 1

n_runs = len(mc_run_dirs)
robust_candidates = [g for g, count in protein_counts.items()
                     if count / n_runs > 0.70]

print(f"\nRobust candidates (>70% run inclusion): {len(robust_candidates)}")
for g in sorted(robust_candidates):
    print(f"  {g}  ({protein_counts[g]}/{n_runs} runs)")
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `dataframe_files` directory empty | Step 4 not yet completed | Run Steps 1–4 first |
| `gene_list.txt` not found | File not yet created | Create a plain-text file with one gene name per line |
| Regression returns NaN coefficients | Perfect separation in a small data subset | Exclude proteins with fewer than ~20 residues in one class, or add a regularization term |
| Monte Carlo does not converge | Too few Monte Carlo steps | Increase the number of steps in `MC.run()` |
| All proteins in same group | Insufficient variance in entanglement enrichment | Verify `region` column is correctly assigned across the dataset |
